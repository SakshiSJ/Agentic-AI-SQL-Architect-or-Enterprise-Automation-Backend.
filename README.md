Enterprise NL-to-SQL Agentic PipelineA production-grade Natural Language to SQL (NL-to-SQL) engine designed for industrial logistics (AutoPlant).
This system utilizes LLMs to translate complex business queries into validated, executable MySQL queries using a metadata-driven approach.
Key FeaturesDynamic Table Routing: Automatically identifies the most authoritative database table based on semantic metadata (chatgpt_engine.py).Metadata-Driven Prompting: Injects table schemas, business terms, and enum constraints directly into the LLM context to ensure high query precision (sql_prompt.py).Multi-Stage SQL Validation: * SQL Builder: Strips markdown and ensures only SELECT statements are processed.SQL Validator: Cross-references generated identifiers against a strict allowed-column whitelist (sql_validator.py).Semantic Query Rewriting: Maps logical business terms to physical database columns with spaces or specific naming conventions (sql_rewriter.py).Autonomous Error Handling: Designed with "reasoning loops" to ensure valid syntax before execution.
🛠️ Tech StackLanguage: Python 3.xAI Framework: OpenAI API (GPT-4o-mini / GPT-4)Database: MySQLLogic: LangChain-inspired Agentic Workflows, Regex-based SQL Parsing📂 Project StructureFileDescriptionmain.pyThe central entry point for the chatbot loop.chatgpt_engine.pyHandles dynamic table selection and LLM communication.structure.jsonThe "Source of Truth" containing table schemas and business logic.sql_validator.pyStrict security layer to prevent SQL injection and invalid column access.sql_rewriter.pyNormalizes queries for specific database environments.db.pyManages database connectivity and execution.
⚙️ How it WorksInput: User asks, "How many vehicles are in the yard?"Route: The Router identifies tbl_ap_active_trips as the authoritative source.Generate: The engine maps "yard" to CURRENT STAGE = 'YardIn' using the business_terms metadata.Validate: The validator ensures the SQL doesn't contain forbidden keywords like DROP or DELETE.Execute: The clean SQL is executed against the database, returning real-time operational data.
